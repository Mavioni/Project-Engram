# Engram internals

Short, plain-English explainers for each subsystem. Written for **you** (Mavioni) to understand what Claude's talking about when Claude mentions a piece — and for any future Claude session to orient quickly.

Read these like a map, not a textbook. Pick the one you need.

- **`architecture.md`** — the 30-second tour of how the whole app fits together.
- **`store.md`** — how the app remembers things.
- **`theme.md`** — how light and dark modes work.
- **`iris.md`** — how the IRIS assessment flows through the app.
- **`engram.md`** — how the replica / arena / XP / seals work.
- **`routes.md`** — what each URL in the app shows.
- **`deploy.md`** — how code becomes a live website.

None of these are required reading. They're here when curiosity or confusion hits.
